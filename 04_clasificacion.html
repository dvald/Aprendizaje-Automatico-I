GridSearchCV: Búsqueda de hiperparámetros

GridSearchCV prueba todas las combinaciones de una rejilla de hiperparámetros y evalúa cada una con validación cruzada (k-fold) sobre el conjunto de entrenamiento. Se elige la combinación con mejor accuracy media en CV y se evalúa en un conjunto de test independiente que no se ha tocado durante la búsqueda.

Proceso paso a paso:
  1. Se reserva un 20% de test, que no se usa en toda la búsqueda.
  2. El 80% restante se divide en k folds estratificados por clase.
  3. Para cada combinación de hiperparámetros se hacen k entrenamientos: se entrena con k-1 folds y se valida con el fold restante. Se promedian las k accuracies → accuracy CV.
  4. Se elige la combinación con mejor accuracy CV y se reentrena con todo el 80%.
  5. Se reporta la accuracy sobre el 20% de test, que es una estimación honesta del rendimiento.
Por qué importa: si eligiéramos los hiperparámetros mirando directamente el conjunto de test, estaríamos "contaminando" la evaluación — el modelo estaría optimizado para ese test concreto. La validación cruzada simula varios test sets dentro del training, y deja un test final intocado para la estimación real.

Regresión Logística

La regresión logística es un algoritmo de clasificación que usa la función sigmoide para mapear cualquier entrada a una probabilidad entre 0 y 1. Usa un umbral (threshold) para tomar la decisión de clasificación.

Función Sigmoide y Clasificación

Cómo funciona: Arrastra el umbral para ver cómo cambia la frontera de decisión. Si la probabilidad sigmoide está por debajo del umbral, se clasifica como Negativo (rojo). Si está por encima, se clasifica como Positivo (azul).

K-Nearest Neighbors (KNN)

KNN es un algoritmo no paramétrico que clasifica un punto basándose en los K vecinos más cercanos. Haz clic en el canvas para añadir puntos de entrenamiento (azul y rojo). Luego, haz clic en cualquier lugar con el botón derecho para clasificar un nuevo punto.

Instrucciones:
  • Click izquierdo: añade punto de la clase actual (azul o rojo)
  • Botón "Añadir Clase...": cambia entre clases
  • Click derecho: clasifica el punto donde haces click
  • Usa el slider para cambiar K y observa cómo cambia la clasificación

Árboles de Decisión

Los árboles de decisión dividen recursivamente el espacio de características basándose en criterios de ganancia de información (impureza de Gini). Cada nodo interno contiene una pregunta, y cada hoja es una clasificación.

Construcción paso a paso

Impureza de Gini: Mide la probabilidad de clasificar incorrectamente un elemento seleccionado aleatoriamente. Un nodo con Gini=0 es puro (solo una clase).

Ejemplo de cálculo de Gini

Gini = 1 - Σ(p_i)²

Donde p_i es la proporción de muestras de cada clase en el nodo.

Ejemplo: Nodo con 60% clase A y 40% clase B:
Gini = 1 - (0.6² + 0.4²) = 1 - (0.36 + 0.16) = 0.48

Support Vector Machine (SVM)

SVM busca el hiperplano que maximiza el margen (distancia mínima a los puntos más cercanos de ambas clases). El parámetro C controla cuánto penaliza los errores de clasificación.

SVM con diferentes kernels

Parámetro C: Valores altos (C > 1) penalizan más los errores, creando márgenes más pequeños pero ajuste más preciso. Valores bajos (C < 1) toleran más errores para márgenes más amplios.

Resumen:
  • Regresión Logística: Rápida y eficiente para datos linealmente separables.
  • KNN: Flexible pero lento con muchos datos. Excelente para problemas no lineales pequeños.
  • Árbol de Decisión: Fácil de interpretar pero propenso a sobreajuste.
  • SVM: Potente con kernel no lineal. Excelente para datos complejos.
  • Naive Bayes: Muy rápido, asume independencia entre features. Funciona sorprendentemente bien en muchos casos.
  • Random Forest: Ensemble de árboles con bagging. Reduce el sobreajuste y suele dar fronteras robustas.

Naive Bayes

Naive Bayes aplica el teorema de Bayes asumiendo independencia entre características. Para cada punto x calcula la probabilidad posterior P(C|x) ∝ P(x|C)·P(C) y predice la clase con mayor posterior. La variante Gaussiana modela cada característica con una distribución normal por clase.

Clasificador Naive Bayes Gaussiano (3 clases)

Cómo leer el gráfico: los círculos punteados son las elipses a 1σ y 2σ de cada gaussiana. El color de fondo indica la clase con mayor probabilidad posterior (la ganadora). Ajusta los priors P(A) y P(B) y observa cómo la frontera se desplaza: dar más peso a una clase "invade" zonas de las otras aunque los datos no hayan cambiado.

Teorema de Bayes aplicado

P(C|x) = P(x|C) · P(C) / P(x)

El clasificador asigna: ŷ = argmaxC P(x|C) · P(C)

Para Gaussian Naive Bayes:
P(x|C) = (1 / √(2π·σ²)) · exp(-(x - μ)² / (2σ²))

Random Forest (Bosques de decisión)

Un bosque aleatorio combina múltiples árboles de decisión entrenados sobre muestras bootstrap distintas de los datos originales (bagging). Cada árbol además elige aleatoriamente un subconjunto de características en cada división. La predicción final se obtiene por voto mayoritario de todos los árboles, lo que reduce la varianza respecto a un árbol individual.

Ensemble de árboles con bootstrap

Lo que estás viendo: la intensidad del color representa la proporción de árboles que votan por cada clase. Zonas claras = poco consenso (cerca de la frontera); zonas saturadas = casi todos los árboles coinciden. Aumenta el número de árboles y verás cómo la frontera se suaviza y la precisión se estabiliza.

Hiperparámetros clave

n_estimators: número de árboles (más = más estable, pero más lento).
max_depth: profundidad máxima de cada árbol (controla el sobreajuste individual).
max_features: número de características candidatas por split (introduce diversidad).
bootstrap: si se usa muestreo con reemplazo (activado por defecto).
